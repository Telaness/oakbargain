# コーディング規約

## 必須

- フォーマッタ、リンタ、コンパイラチェック、ユニットテストをすべて通してからprを行ってください。
- `let`は使わずに`const`のみでコーディングをしてください。
- ステータスの型定義には文字列リテラルのユニオン型を使用してください。
- Array.prototype.forEachは使わずに、Array.prototype.mapを使ってください。
- 開発はNext.js×TypeScriptのベストプラクティスに沿って実装してください。
- 実装の共通部分は必ず共通化して、リファクタリングしてください。

## 推奨
- `any`型の使用は避けてください。
- `useEffect`の乱用は避けてください。
- その他、Next×TypeScriptのベストプラクティスに沿って実装してください。
