# OAK BARGAIN 公式HP　要件定義書

**バージョン：** v1.0

---

## 1. 機能要件

### FR-001　ホコリ払い演出（Dust Reveal）

**概要**
サイト初回ロード時、全体にホコリ（パーティクル）がかかっており、マウスムーブ（PC）またはタッチ（SP）操作によってホコリが払われ、コンテンツが徐々に現れる。

**詳細仕様**

- パーティクル数：PC 3,000〜5,000個、SP 1,500〜2,000個
- マウス半径：120px以内のパーティクルが吹き飛ぶ
- パーティクル色：セピア・茶・グレーの混在（ヴィンテージ感）
- 全体の70%以上払った場合、残りパーティクルが自動消滅（フェードアウト 2秒）
- 払ったエリアは再度パーティクルが発生しない

**演出意図**
「埋もれていた価値を見つける」「時を経たものに触れる」ヴィンテージの発掘感・出会い感を表現する。

---

### FR-002　ロゴ回転降下アニメーション

**概要**
OAK BARGAINのロゴが木の天辺から回転しながら幹を伝って降下するアニメーションを実装する。

**詳細仕様**

- スクロールに連動してロゴが降下する（スクロール量とY座標が比例）
- 降下中はY軸回転アニメーションが継続する（Three.jsメッシュ）
- 各LINEセクションに到達するとロゴが一瞬輝くBloomエフェクトを発火
- モバイルではスワイプに連動

---

### FR-003　大樹ビジュアル（Three.js 3D）

**概要**
1本の大樹を3Dで表現し、画面中央に縦長に配置する。木の各部位（花・葉・枝・幹）にLINEが紐づく。

**詳細仕様**

- 幹：太く力強い。テクスチャはリアル系バーク（樹皮）
- 枝：複数本、不規則に広がる
- 葉：葉全体で9割、ボリューム感を重視
- 花：全体の1割、葉の間に点在。Luxury Line対応
- カメラ：初期は木全体俯瞰、スクロールで徐々に降下・ズームイン
- ライティング：月光・夕暮れをイメージした間接照明トーン

**デザイン方針**
「古さ＝弱さではなく、時を重ねたからこその重厚感・存在感・生命力」を表現する。細い木・弱そうな印象は避け、歳を重ねた力強い大樹をビジュアルの中心に据える。

---

### FR-004　LINEセクション（4ライン詳細）

**概要**
木の各部位をクリック（タップ）すると対応するLINEの詳細ページへ遷移する。

**詳細仕様**

- ホバー時：対象部位が光る・揺れる演出
- クリック時：GSAPカーテンアニメーションでページ遷移
- 各LINEページ：ライン名・コンセプト・価格帯・イメージビジュアル（絵画風）・戻るボタン
- グレードの上昇は色調・フォントサイズ・エフェクトの豪華さで表現

---

### FR-005　絵画風画像演出

**概要**
商品・雰囲気画像は普通の写真ではなく、絵画のような質感でフィルタリングして表示する。

**詳細仕様**

- Three.jsのpost-processingでオイルペイント・ビネット・セピアフィルター適用
- または画像自体をCSSフィルター（sepia / contrast / saturate）で加工
- ブランドのトーンに合わせたアンバー・ゴールド系の色味

## 2. 非機能要件

| カテゴリ | 指標 | 目標値 |
|----------|------|--------|
| パフォーマンス | LCP | 3秒以内（PC）/ 5秒以内（SP） |
| パフォーマンス | FID | 100ms以内 |
| パフォーマンス | CLS | 0.1以下 |
| 対応環境 | ブラウザ | Chrome / Safari / Firefox / Edge（最新2バージョン） |
| 対応環境 | デバイス | PC（1280px〜）/ タブレット（768px〜）/ SP（375px〜） |
| アクセシビリティ | WCAG | WCAG 2.1 Level A準拠（最低限） |
| SEO | メタ情報 | OGP・Twitter Card・JSON-LD設定 |
| セキュリティ | HTTPS | Vercel標準SSL証明書 |
| 保守性 | コード品質 | ESLint + Prettier + TypeScript strict mode |

---

## 3. 制約条件

- オンライン販売機能は実装しない（EC機能不要）
- 商品在庫管理システムとの連携は対象外
- CMSはこの段階では実装しない（静的コンテンツで対応）
- Three.jsのWebGL非対応ブラウザにはフォールバック画面を表示
